# 管理员端动作库接口说明

路由定义在 `api/admin_action.py`，经 `main.py` 挂载后**统一前缀**为 `/api`，本模块路由前缀为 `/admin/actions`，即完整路径形如 `/api/admin/actions/...`。

**鉴权**：所有接口均需在请求头携带 JWT：

`Authorization: Bearer <token>`

且 Token 载荷中的 **`role` 须为 `admin`**。非管理员调用返回 **403**，`message` 为「无权限」。

---

## 统一响应格式

与全项目一致（`utils/response.py`）：

| 场景 | HTTP 状态码 | Body 结构 |
|------|-------------|-----------|
| 成功 | 200 | `{ "code": 200, "message": "提示文案", "data": ... }` |
| 业务失败 | 与 `code` 一致（如 400/403/404） | `{ "code": <数字>, "message": "错误说明", "data": null }` |

参数校验失败时可能返回 **422**（FastAPI 默认校验响应）。

---

## 动作状态说明（`status`）

管理员列表与详情中出现的 **`status`** 含义如下（与用户端、教练端文档对齐，并补充下架态）：

| 值 | 含义 |
|:--:|------|
| `0` | 待审核 |
| `1` | 已上线（用户端动作库可见） |
| `2` | 已驳回 |
| `3` | 已下架（由管理员对「已上线」动作执行下架后） |

**说明**：列表与审核逻辑仅包含**未软删**记录（`is_deleted == 0`）；已软删数据在 `get_action_by_id` 中不可查，会按「不存在」处理。

---

## 接口索引

| 序号 | 说明 | 方法与路径 |
|:----:|------|------------|
| 1 | 动作列表（可按状态筛选、分页） | `GET` `/api/admin/actions` |
| 2 | 单条动作完整详情（预览，不限状态） | `GET` `/api/admin/actions/{action_id}/detail` |
| 3 | 审核（通过 / 驳回） | `PUT` `/api/admin/actions/{action_id}/review` |
| 4 | 下架已上线动作 | `PUT` `/api/admin/actions/{action_id}/offline` |

---

## 1. 动作列表

| 项目 | 内容 |
|------|------|
| 路径 | `GET /api/admin/actions` |
| 作用 | 分页返回全部投稿摘要；可按 `status` 筛选；含作者昵称；按创建时间**降序**（与 `get_admin_actions` 一致） |

**Query 参数**

| 参数 | 类型 | 必填 | 默认 | 说明 |
|------|------|:----:|------|------|
| `status` | integer | 否 | — | 筛选状态：`0` 待审核 / `1` 已上线 / `2` 已驳回 / `3` 已下架；不传则返回全部（未软删） |
| `page` | integer | 否 | `1` | 页码 |
| `page_size` | integer | 否 | `20` | 每页条数 |

**成功时 `data`**

| 字段 | 说明 |
|------|------|
| `list` | 数组，单项见下表 |
| `total` | 总条数 |
| `page` | 当前页 |
| `page_size` | 每页条数 |

**`list` 单项字段**

| 字段 | 说明 |
|------|------|
| `id` | 动作 ID |
| `title` | 标题 |
| `body_part` | 训练部位 |
| `category` | 动作类型 |
| `difficulty` | 难度 |
| `status` | 状态（见上文状态表） |
| `reject_reason` | 驳回原因 |
| `author_id` | 投稿教练用户 ID |
| `author_name` | 投稿教练昵称；无则 `""` |
| `created_at` | 创建时间 `YYYY-MM-DD HH:MM:SS` |

---

## 2. 单条详情（预览）

| 项目 | 内容 |
|------|------|
| 路径 | `GET /api/admin/actions/{action_id}/detail` |
| 作用 | 返回单条动作**完整内容**，**不区分状态**（待审核/已驳回/已下架均可查）；**不会**增加 `view_count`（与用户端详情不同） |

**路径参数**

| 参数 | 类型 | 说明 |
|------|------|------|
| `action_id` | integer | 动作 ID |

**成功时 `data`**

| 字段 | 说明 |
|------|------|
| `id` | 动作 ID |
| `title` | 标题 |
| `body_part` | 训练部位 |
| `category` | 动作类型 |
| `difficulty` | 难度 |
| `cover_img` | 封面相对路径；静态访问 `{根}/api/static/{path}` |
| `video_url` | 视频相对路径 |
| `description` | 文字说明 |
| `steps` | 步骤数组 |
| `cautions` | 注意事项数组 |
| `status` | 当前状态 |
| `reject_reason` | 驳回原因 |
| `author_name` | 作者昵称 |
| `created_at` | 创建时间 `YYYY-MM-DD HH:MM:SS` |

**业务错误**：记录不存在或已软删（404）。

---

## 3. 审核（通过 / 驳回）

| 项目 | 内容 |
|------|------|
| 路径 | `PUT /api/admin/actions/{action_id}/review` |
| 作用 | 仅当动作当前为 **待审核（`status == 0`）** 时可审核：`1` 通过上线，`2` 驳回并写入原因；并记录审核人、审核时间（见 `CRUD.review_action`） |

**路径参数**

| 参数 | 类型 | 说明 |
|------|------|------|
| `action_id` | integer | 动作 ID |

**请求体字段**（JSON）

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `status` | integer | 是 | 仅允许 `1`（通过）或 `2`（驳回） |
| `reject_reason` | string | 条件 | 驳回（`status == 2`）时**必填**且不能为空；通过时可为空 |

**成功时 `data`**：`null`；`message` 为「已通过」或「已驳回」。

**常见业务错误**：非管理员（403）、`status` 非 1/2（400）、驳回未填原因（400）、动作不存在（404）、当前非待审核（400）。

---

## 4. 下架已上线动作

| 项目 | 内容 |
|------|------|
| 路径 | `PUT /api/admin/actions/{action_id}/offline` |
| 作用 | 仅当动作为 **已上线（`status == 1`）** 时，将 `status` 改为 **`3`（已下架）**；用户端列表仅展示 `status == 1`，下架后用户不可见 |

**路径参数**

| 参数 | 类型 | 说明 |
|------|------|------|
| `action_id` | integer | 动作 ID |

**请求**：无 Body。

**成功时 `data`**：`null`，`message` 一般为「已下架」。

**常见业务错误**：非管理员（403）、动作不存在（404）、当前非已上线（400）。

---

## 补充说明

1. **与用户端、教练端分工**：用户端列表/详情见 `api/action.py`；教练投稿与修改见 `api/coach_action.py`；管理员侧以本文档为准。
2. **静态资源**：`cover_img`、`video_url` 仍为相对路径，经 **`GET /api/static/{相对路径}`** 访问。
3. **驳回后教练侧**：教练可依据教练端接口修改被驳回动作并再次进入待审核（`status` 回到 `0`），流程见 `docs/教练端动作库接口.md`。
