# 课程管理与预约实现文档

---

## 【任务书】

**1. 需求说明：**
实现课程管理与预约模块，支持三条业务线：
- 教练端：发布课程（提交审核）、管理自己的课程、查看课程预约列表、同意/拒绝预约申请
- 管理员端：审核课程（通过/驳回）、下架已上线课程
- 用户端：浏览已上线课程列表、申请预约、查看我的预约、取消预约

**2. 实现思路：**
参考动作库/饮食文章模块，分三层：`schemas/course.py` → `CRUD/course.py` → 三个路由文件。
课程新增"审核"流程（原模型 status 字段需扩展），预约新增"教练审批"流程（Reservation.status 扩展）。

**3. 前端要做什么：**
- 用户端：课程列表（筛选+分页）、课程详情卡片、申请预约、我的预约列表（含取消）
- 教练端：发布课程表单（含封面上传）、我的课程列表（含编辑/下架）、课程预约申请列表（同意/拒绝）
- 管理员端：课程审核列表（状态筛选）、详情预览、通过/驳回/下架

**4. 后端要做什么：**
- `schemas/course.py`：CourseCreate / CourseUpdate / CourseReview / ReservationAction
- `CRUD/course.py`：课程增删改查 + 审核 + 下架；预约创建/查询/审批/取消
- `api/course.py`：用户端只读 + 预约接口（需登录）
- `api/coach_course.py`：教练端课程管理 + 预约审批（需 coach 角色）
- `api/admin_course.py`：管理员审核接口（需 admin 角色）
- `main.py`：注册三个新路由

**5. 数据表结构：**
已有 `courses` 表和 `reservations` 表，需扩展 status 字段含义（见下文）。

## 一、数据模型（已有，需了解字段扩展）

## 二、Course status 状态流转

```
教练发布
    ↓
[0] 待审核
    ↓ 管理员审核
   ↙         ↘
[1] 招募中   [4] 已驳回
    ↓              ↓ 教练修改后重新提交
[2] 满员        [0] 待审核（循环）
    ↓
[3] 已结束
[5] 已下架（管理员对招募中/满员执行）
```

| status | 含义 | 用户端可见 | 教练可编辑 | 教练可删除 |
|:------:|------|:----------:|:----------:|:----------:|
| 0 | 待审核 | 否 | 否 | 是 |
| 1 | 招募中 | 是 | 否 | 否 |
| 2 | 满员 | 是（不可预约） | 否 | 否 |
| 3 | 已结束 | 是（只读） | 否 | 否 |
| 4 | 已驳回 | 否 | 是 | 是 |
| 5 | 已下架 | 否 | 否 | 否 |

## 三、Reservation status 状态流转

```
用户申请预约
    ↓
[1] 待审批
    ↓ 教练操作
   ↙         ↘
[2] 已确认   [3] 已拒绝
    ↓
[0] 已取消（用户主动取消，仅限待审批/已确认状态）
```

| status | 含义 | enrolled 变化 |
|:------:|------|:-------------:|
| 0 | 已取消 | 若从已确认取消则 -1 |
| 1 | 待审批 | 不变 |
| 2 | 已确认 | +1 |
| 3 | 已拒绝 | 不变 |

### 课程相关

| 函数 | 说明 |

### 预约相关

| 函数 | 说明 |
|------|------|
| `get_reservation(db, user_id, course_id)` | 查询用户对某课程的预约记录 |

## 六、API 路由设计

### 6.1 用户端 `api/course.py`

路由前缀：`/courses`，挂载后：`/api/courses`

**鉴权**：列表无需登录；预约相关需登录（任意角色）

#### GET `/api/courses` — 课程列表

Query 参数：

| 参数 | 类型 | 必填 | 默认 | 说明 |
|------|------|:----:|------|------|
| `category` | string | 否 | — | 分类精确匹配 |
| `keyword` | string | 否 | — | 标题模糊搜索 |
| `page` | integer | 否 | 1 | 页码 |
| `page_size` | integer | 否 | 12 | 每页条数 |

成功 `data`：

```json
{
  "list": [
    {
      "id": 1,
      "title": "燃脂塑形课",
      "category": "有氧",
      "difficulty": 2,
      "cover_img": "course_covers/xxx.jpg",
      "price": "99.00",
      "max_people": 10,
      "enrolled": 7,
      "start_time": "2026-04-20 09:00:00",
      "location": "健身房A区",
      "status": 1,
      "coach_name": "王教练"
    }
  ],
  "total": 20,
  "page": 1,
  "page_size": 12
}
```

#### GET `/api/courses/{course_id}` — 课程详情

返回完整字段（含 description），不限 status=1/2 均可查（满员也能看详情）。

#### POST `/api/courses/{course_id}/reserve` — 申请预约

需登录。

业务校验：
- 课程 status 不为 1（招募中）→ 400
- 已有未取消的预约记录 → 400
- enrolled >= max_people → 400（满员）

成功 `data`：`{ "reservation_id": 新建预约ID }`

#### GET `/api/courses/my-reservations` — 我的预约列表

需登录。返回当前用户全部预约记录，含课程基本信息。

成功 `data`（数组）：

| 字段 | 说明 |
|------|------|
| `reservation_id` | 预约 ID |
| `course_id` | 课程 ID |
| `course_title` | 课程名称 |
| `coach_name` | 教练名 |
| `start_time` | 开始时间 |
| `status` | 预约状态：0已取消 1待审批 2已确认 3已拒绝 |
| `cancel_reason` | 拒绝/取消原因 |
| `created_at` | 预约时间 |

#### DELETE `/api/courses/reservations/{reservation_id}` — 取消预约

需登录，仅限本人，status 为 1（待审批）或 2（已确认）时可取消。

返回当前教练未软删的全部课程，按创建时间降序。

| 字段 | 说明 |

仅限本人且 `status` 为 0（待审核）或 4（已驳回）时可删，执行软删除。

| `status` | 预约状态 |
| `cancel_reason` | 拒绝原因 |

业务校验：
- 确认时若 enrolled >= max_people → 400（已满员）
- 仅 status=1（待审批）的预约可审批

成功：`data: null`，`message: "已确认"` 或 `"已拒绝"`

## 十、完整业务流程

```
1. 教练发布课程（status=0 待审核）
      ↓
2. 管理员审核通过（status=1 招募中）
      ↓
3. 用户端可见，用户申请预约（reservation.status=1 待审批）
      ↓
4. 教练确认预约（reservation.status=2，course.enrolled+1）
      ↓
   enrolled == max_people 时 course.status 自动变为 2（满员）
      ↓
5. 课程结束后 status=3（已结束）/ 管理员可随时下架 status=5

用户可在待审批(1)或已确认(2)状态下取消预约
  → 取消已确认预约时 course.enrolled-1（若课程为满员则恢复为招募中）
```

## 十一、与动作库/饮食文章的主要差异

| 差异点 | 动作库/文章 | 课程 |
|--------|------------|------|
| 审核通过后状态 | status=1（固定） | status=1（招募中），可变为2/3 |
| 驳回状态值 | 2 | 4（避免与满员2冲突） |
| 下架状态值 | 3 | 5 |
| 额外业务 | 无 | 预约申请 + 教练审批 + 人数管理 |
| 封面存储目录 | diet_covers/ | course_covers/ |
