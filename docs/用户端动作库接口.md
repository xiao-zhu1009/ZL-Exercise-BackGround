# 用户端动作库接口说明

路由定义在 `api/action.py`，经 `main.py` 挂载后**统一前缀**为 `/api`，本模块路由前缀为 `/actions`，即完整路径形如 `/api/actions`、`/api/actions/{id}`。

**鉴权**：用户端动作库**无需登录**；不带 `Authorization` 也可访问。

**数据范围**：列表与详情仅面向**已上线**动作（`status == 1`）且**未软删**（`is_deleted == 0`）。列表按**浏览量降序**排序。

---

## 统一响应格式

与全项目一致（`utils/response.py`）：

| 场景 | HTTP 状态码 | Body 结构 |
|------|-------------|-----------|
| 成功 | 200 | `{ "code": 200, "message": "success", "data": ... }` |
| 业务失败 | 与 `code` 一致（如 404） | `{ "code": <数字>, "message": "错误说明", "data": null }` |

列表与详情均为 **GET**，无 JSON Body；Query 参数见各节。

---

## 接口索引

| 序号 | 说明 | 方法与路径 |
|:----:|------|------------|
| 1 | 动作列表（筛选 + 分页） | `GET` `/api/actions` |
| 2 | 动作详情（浏览数 +1） | `GET` `/api/actions/{action_id}` |

---

## 1. 动作列表

| 项目 | 内容 |
|------|------|
| 路径 | `GET /api/actions` |
| 作用 | 分页返回已上线动作卡片所需字段；支持部位、类型、难度、名称关键词筛选 |

**Query 参数**

| 参数 | 类型 | 必填 | 默认 | 说明 |
|------|------|:----:|------|------|
| `body_part` | string | 否 | — | 训练部位，如与库中数据一致则精确匹配 |
| `category` | string | 否 | — | 动作类型 |
| `difficulty` | integer | 否 | — | 难度（与库中存储一致，如 1/2/3） |
| `keyword` | string | 否 | — | 标题模糊包含 |
| `page` | integer | 否 | `1` | 页码 |
| `page_size` | integer | 否 | `12` | 每页条数 |

**成功时 `data`**

| 字段 | 说明 |
|------|------|
| `list` | 数组，每项见下表 |
| `total` | 符合条件的总条数 |
| `page` | 当前页 |
| `page_size` | 每页条数 |

**`list` 单项字段**

| 字段 | 说明 |
|------|------|
| `id` | 动作 ID |
| `title` | 标题 |
| `body_part` | 训练部位 |
| `category` | 动作类型 |
| `difficulty` | 难度 |
| `cover_img` | 封面相对路径；无图时可能为 `null`。前端访问示例：`{API_BASE}/static/{cover_img}` |
| `view_count` | 浏览次数 |

---

## 2. 动作详情

| 项目 | 内容 |
|------|------|
| 路径 | `GET /api/actions/{action_id}` |
| 作用 | 返回完整展示字段；**每次成功返回前**会将该动作 `view_count` **加 1**（先校验再自增） |

**路径参数**

| 参数 | 类型 | 说明 |
|------|------|------|
| `action_id` | integer | 动作 ID |

**成功时 `data`**

| 字段 | 说明 |
|------|------|
| `id` | 动作 ID |
| `title` | 标题 |
| `body_part` | 训练部位 |
| `category` | 动作类型 |
| `difficulty` | 难度 |
| `cover_img` | 封面相对路径；图片 URL 拼接方式同列表 |
| `video_url` | 视频相对路径；拼接示例：`{API_BASE}/static/{video_url}` |
| `description` | 文字说明 |
| `steps` | 步骤列表（JSON 数组；无则为 `[]`） |
| `cautions` | 注意事项列表（JSON 数组；无则为 `[]`） |
| `view_count` | 浏览次数（**已为 +1 后的值**） |
| `author_name` | 作者展示名（取用户昵称；无作者信息时为空字符串） |
| `created_at` | 创建时间，格式 `YYYY-MM-DD HH:MM:SS` |

**业务错误**：动作不存在、未上线或已软删时返回 **404**，`message` 为「动作不存在」。

---

## 补充说明

1. **静态资源**：`cover_img`、`video_url` 一般为相对路径，需与后端挂载的静态目录一致；项目中将静态资源挂在 **`/api/static`** 下时，完整 URL 为 **`{协议+主机+端口}/api/static/{相对路径}`**（与前端 `ActionLibrary` 中拼接方式一致即可）。
2. **教练投稿、管理员审核**等接口不在本文档范围，见 `api/coach_action.py`、`api/admin_action.py` 等。
