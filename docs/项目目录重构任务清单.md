### Task 2 — 新建 `CRUD/` 目录，抽离路由中的数据库操作

#### 2.1 新建 `CRUD/user.py`

从 `api/auth.py` 和 `api/user.py` 中抽离以下函数：

| 函数名 | 来源路由 | 功能 |
|--------|----------|------|
| `get_user_by_account(db, account)` | `api/auth.py` login | 按账号或手机号查用户 |
| `get_user_by_id(db, user_id)` | `api/user.py` get_profile | 按 ID 查用户 |
| `create_user(db, form)` | `api/auth.py` register | 创建新用户 |
| `update_user_profile(db, user, form)` | `api/user.py` update_profile | 更新昵称/手机/签名 |
| `update_user_password(db, user, new_pwd)` | `api/user.py` update_password | 更新密码 |
| `check_user_exists(db, username, phone)` | `api/auth.py` register | 检查账号/手机是否重复 |

**具体文件变更：**
- [ ] 新建 `CRUD/__init__.py`（空文件）
- [ ] 新建 `CRUD/user.py`，实现上表所有函数，函数签名显式传入 `AsyncSession`

#### 2.2 新建 `CRUD/body_stats.py`

从 `api/user.py` 中抽离以下函数：

| 函数名 | 来源路由 | 功能 |
|--------|----------|------|
| `get_body_stats(db, user_id)` | `api/user.py` get_body_stats | 查询身体指标 |
| `upsert_body_stats(db, user_id, form)` | `api/user.py` update_body_stats | 新增或更新身体指标，自动计算 BMI/WHR |

**具体文件变更：**
- [ ] 新建 `CRUD/body_stats.py`，实现上表所有函数

### Task 3 — 重构 `api/auth.py`，路由只调用 CRUD

**改造前（当前）：** 路由内直接写 `db.execute(select(User)...)`

**改造后（目标）：** 路由只做参数校验 + 调用 CRUD + 返回响应

- [ ] 修改 `api/auth.py`：
  - 删除 `from sqlalchemy import select, or_`
  - 删除 `from models.user import User`
  - 新增 `from CRUD.user import get_user_by_account, create_user, check_user_exists`
  - login 路由：调用 `get_user_by_account()` 替代内联 SQL
  - register 路由：调用 `check_user_exists()` + `create_user()` 替代内联 SQL

---

### Task 4 — 重构 `api/user.py`，路由只调用 CRUD

- [ ] 修改 `api/user.py`：
  - 删除 `from sqlalchemy import select`

## 四、重构后分层调用关系

```
HTTP 请求
  → api/ 路由（schemas 校验入参，deps 鉴权）
  → CRUD/（AsyncSession + models 做增删改查）
  → models/（ORM 定义）
  ← api/ 路由调用 utils/response.py 包装统一响应
  ← 返回 { "code": int, "message": str, "data": ... }
```

## 五、各层职责速查

| 目录 | 职责 | 禁止 |
|------|------|------|
| `config/` | 环境变量、常量配置 | 业务逻辑、SQL |
| `models/` | ORM 类定义 | Pydantic、业务逻辑 |
| `schemas/` | Pydantic 请求/响应模型 | ORM、数据库操作 |
| `CRUD/` | 单表增删改查封装 | 路由装饰器、HTTP 响应 |
| `api/` | 路由、鉴权、调用 CRUD | 直接写 SQL、长业务逻辑 |
| `utils/` | JWT、响应封装等通用工具 | 单表专属 CRUD |
| `db/` | 引擎、Session、Base | 业务逻辑 |
