# 教练端动作库接口说明

路由定义在 `api/coach_action.py`，经 `main.py` 挂载后**统一前缀**为 `/api`，本模块路由前缀为 `/coach/actions`，即完整路径形如 `/api/coach/actions/...`。

**鉴权**：所有接口均需在请求头携带 JWT：

`Authorization: Bearer <token>`

且 Token 载荷中的 **`role` 须为 `coach`**（与数据库用户角色一致，见 `get_current_user`）。非教练角色调用任意本组接口时返回 **403**，`message` 为「无权限」。

---

## 统一响应格式

与全项目一致（`utils/response.py`）：

| 场景 | HTTP 状态码 | Body 结构 |
|------|-------------|-----------|
| 成功 | 200 | `{ "code": 200, "message": "提示文案", "data": ... }` |
| 业务失败 | 与 `code` 一致（如 400/403/404） | `{ "code": <数字>, "message": "错误说明", "data": null }` |

参数校验失败时由框架返回 **422**（与业务信封格式可能不同，属 FastAPI 默认行为）。

---

## 动作状态说明（`status`）

| 值 | 含义 |
|:--:|------|
| `0` | 待审核 |
| `1` | 已通过（上线） |
| `2` | 已驳回 |

教练端**修改**仅允许针对 **已驳回（2）**；**删除**仅允许 **待审核（0）** 或 **已驳回（2）**。

---

## 接口索引

| 序号 | 说明 | 方法与路径 |
|:----:|------|------------|
| 1 | 上传封面图 | `POST` `/api/coach/actions/upload/cover` |
| 2 | 上传教学视频 | `POST` `/api/coach/actions/upload/video` |
| 3 | 发布新动作（待审核） | `POST` `/api/coach/actions` |
| 4 | 我的投稿列表 | `GET` `/api/coach/actions` |
| 5 | 修改被驳回的动作并重新提交 | `PUT` `/api/coach/actions/{action_id}` |
| 6 | 删除待审核或已驳回的动作 | `DELETE` `/api/coach/actions/{action_id}` |

## 1. 上传封面图

| 项目 | 内容 |
|------|------|
| 路径 | `POST /api/coach/actions/upload/cover` |
| 作用 | 校验格式与大小后写入 `static/action_covers/`，返回**相对路径**，供发布/修改动作时填入 `cover_img` |

**请求**

| 项目 | 说明 |
|------|------|
| `Content-Type` | `multipart/form-data` |
| 表单字段 | `file`：图片文件（必填） |

**限制**：MIME 须为 `image/jpeg`、`image/png`、`image/webp`；单文件不超过 **5MB**；文件名形如 `{教练user_id}_{uuid}{后缀}`。

**成功时 `data`**

| 字段 | 说明 |
|------|------|
| `path` | 相对路径，如 `action_covers/{文件名}`；完整访问 URL 为 `{服务根}/api/static/{path}` |

**常见业务错误**：非教练（403）、格式不符（400）、超过 5MB（400）。

---

## 2. 上传教学视频

| 项目 | 内容 |
|------|------|
| 路径 | `POST /api/coach/actions/upload/video` |
| 作用 | 分块写入 `static/action_videos/`，返回**相对路径**，供填入 `video_url` |

**请求**

| 项目 | 说明 |
|------|------|
| `Content-Type` | `multipart/form-data` |
| 表单字段 | `file`：视频文件（必填） |

**限制**：MIME 须为 `video/mp4` 或 `video/webm`；总大小不超过 **200MB**；文件名形如 `{教练user_id}_{uuid}{后缀}`。

**成功时 `data`**

| 字段 | 说明 |
|------|------|
| `path` | 相对路径，如 `action_videos/{文件名}`；完整 URL 拼接方式同封面 |

**常见业务错误**：非教练（403）、格式不符（400）、超过 200MB（400，服务端会删除未写完的临时文件）。

---

## 3. 发布新动作

| 项目 | 内容 |
|------|------|
| 路径 | `POST /api/coach/actions` |
| 作用 | 创建一条动作记录，初始 **`status = 0`（待审核）** |

**请求体字段**（JSON，`Content-Type: application/json`）

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `title` | string | 是 | 动作标题 |
| `body_part` | string | 是 | 训练部位，如：胸/背/腿/肩/手臂/核心 |
| `category` | string | 是 | 动作类型，如：力量/有氧/拉伸/综合 |
| `difficulty` | integer | 否 | 默认 `1`：1 初级 / 2 中级 / 3 高级 |
| `cover_img` | string | 否 | 默认 `""`；一般为上传封面接口返回的 `path` |
| `video_url` | string | 否 | 默认 `""`；一般为上传视频接口返回的 `path` |
| `description` | string | 否 | 文字说明 |
| `steps` | string[] | 否 | 步骤列表 |
| `cautions` | string[] | 否 | 注意事项列表 |

**成功时 `data`**

| 字段 | 说明 |
|------|------|
| `id` | 新建动作 ID |

---

## 4. 我的投稿列表

| 项目 | 内容 |
|------|------|
| 路径 | `GET /api/coach/actions` |
| 作用 | 返回当前登录教练**未软删**的全部投稿，按创建时间**降序**（与 CRUD 一致） |

**请求**：无 Query、无 Body。

**成功时 `data`**：数组，元素结构如下。

| 字段 | 说明 |
|------|------|
| `id` | 动作 ID |
| `title` | 标题 |
| `body_part` | 训练部位 |
| `category` | 动作类型 |
| `difficulty` | 难度 |
| `status` | 状态：0 待审核 / 1 通过 / 2 驳回 |
| `reject_reason` | 驳回原因；未驳回时可能为空字符串 |
| `cover_img` | 封面相对路径，无则 `""` |
| `video_url` | 视频相对路径，无则 `""` |
| `description` | 说明，无则 `""` |
| `steps` | 步骤数组 |
| `cautions` | 注意事项数组 |
| `created_at` | 创建时间 `YYYY-MM-DD HH:MM:SS` |

## 5. 修改被驳回的动作

| 项目 | 内容 |
|------|------|
| 路径 | `PUT /api/coach/actions/{action_id}` |
| 作用 | 仅当动作为**本人**且 **`status == 2`（已驳回）** 时可改；保存后**重置为待审核（0）**并**清空驳回原因** |

**路径参数**

| 参数 | 类型 | 说明 |
|------|------|------|
| `action_id` | integer | 动作 ID |

**请求体字段**（均为可选，只传需要修改的字段）

| 字段 | 类型 | 说明 |
|------|------|------|
| `title` | string | 标题 |
| `body_part` | string | 训练部位 |
| `category` | string | 动作类型 |
| `difficulty` | integer | 难度 |
| `cover_img` | string | 封面相对路径 |
| `video_url` | string | 视频相对路径 |
| `description` | string | 说明 |
| `steps` | string[] | 步骤 |
| `cautions` | string[] | 注意事项 |

**成功时 `data`**：`null`，`message` 一般为「已重新提交审核」。

**常见业务错误**：非教练（403）、非本人或记录不存在（404）、状态不是已驳回（400）。

---

## 6. 删除动作（软删除）

| 项目 | 内容 |
|------|------|
| 路径 | `DELETE /api/coach/actions/{action_id}` |
| 作用 | 仅**本人**且状态为 **待审核（0）** 或 **已驳回（2）** 时可删；执行为**软删除**（`is_deleted = 1`） |

**路径参数**

| 参数 | 类型 | 说明 |
|------|------|------|
| `action_id` | integer | 动作 ID |

**成功时 `data`**：`null`，`message` 一般为「已删除」。

**常见业务错误**：非教练（403）、非本人或不存在（404）、已通过等不可删状态（400）。

---

## 补充说明

1. **推荐流程**：先 `upload/cover`、`upload/video` 取得 `path` → 再 `POST /api/coach/actions` 提交 JSON（或修改时再 `PUT`）。
2. **静态访问**：封面、视频与全站一致，经 `GET /api/static/{path}` 访问（`path` 为接口返回的相对路径）。
3. **管理员审核**接口见 `api/admin_action.py`，本文档不包含。
