# 用户端文章库模块任务书

---

## 【任务书】

**1. 需求说明：**
将教练投稿的饮食文章从"饮食中心"的"推荐餐食" Tab 中移除，在用户端单独新增一个"文章库"模块（独立菜单项 + 独立路由），用于浏览教练投稿并审核通过的文章，支持分类筛选、关键词搜索、分页、点击进入详情。

**2. 实现思路：**
- 后端接口无需改动，复用现有 `GET /api/diet/articles` 和 `GET /api/diet/articles/{id}` 接口
- 前端：
  1. 新增独立页面 `src/views/user/ArticleLibrary/index.vue`（文章列表）
  2. 文章详情复用现有 `ArticleDetail.vue`，路由路径改为 `/user/articles/:id`（原 `/user/diet/articles/:id` 保留兼容）
  3. 饮食中心删除"推荐餐食" Tab 及相关 data/methods
  4. UserLayout 侧边栏新增"文章库"菜单项
  5. router 新增对应路由

**3. 前端要做什么：**
- 新建 `src/views/user/ArticleLibrary/index.vue`：文章卡片列表，分类筛选 + 关键词搜索 + 分页
- 修改 `src/router/index.js`：新增 `/user/articles` 和 `/user/articles/:id` 路由
- 修改 `src/components/Layout/UserLayout.vue`：侧边栏加"文章库"菜单项
- 修改 `src/views/user/DietCenter/index.vue`：删除"推荐餐食" Tab 及其相关逻辑

**4. 后端要做什么：**
无需改动，复用现有接口。

**5. 接口设计（复用现有）：**

```
GET /api/diet/articles              文章列表（status=1，支持 category/keyword/page/page_size）
GET /api/diet/articles/{id}         文章详情（浏览数+1）
```

**6. 数据表结构：**
无需变更。

## 二、路由设计

```
/user/articles          → ArticleLibrary/index.vue   文章库列表
/user/articles/:id      → DietCenter/ArticleDetail.vue  文章详情（复用）
/user/diet/articles/:id → 保留，redirect 到 /user/articles/:id（兼容旧链接）
```

---

## 三、页面结构（ArticleLibrary/index.vue）

```
文章库
├── 筛选栏（分类下拉 + 关键词搜索）
├── 文章卡片列表（3列，封面 + 标题 + 分类 + 浏览数 + 摘要）
├── 空状态
└── 分页
```
