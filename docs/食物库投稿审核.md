# 食物库投稿审核

## 1. 需求说明
教练端可投稿新食物，管理员审核通过后进入系统库，用户端搜索时可见。

## 2. 接口设计
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /coach/foods | 教练投稿食物 |
| GET | /coach/foods | 教练查自己的投稿 |
| DELETE | /coach/foods/{id} | 教练删除待审核/驳回的投稿 |
| GET | /admin/foods | 管理员查投稿列表（?status=） |
| PUT | /admin/foods/{id}/approve | 管理员通过 |
| PUT | /admin/foods/{id}/reject | 管理员驳回 |

## 3. 数据表变更
foods 表新增字段：
- status SMALLINT DEFAULT 0（0=待审核 1=通过 2=驳回）
- reject_reason VARCHAR(255) DEFAULT ''

系统预置食物 status=1，教练投稿 status=0，通过后改为 status=1 且 is_custom=0。

## 4. 搜索逻辑
search_foods 只返回 status=1 的食物。
