# 饮食文章实现文档

---

## 【任务书】

**1. 需求说明：**
实现饮食中心文章模块，支持三条业务线：
- 用户端：浏览已上线的饮食文章（列表 + 详情，无需登录）
- 教练端：投稿文章、管理自己的投稿（增删改）
- 管理员端：审核文章（通过/驳回）、下架已上线文章

**2. 实现思路：**
参考动作库模块，分三层：`schemas/diet_article.py` → `CRUD/diet_article.py` → `api/diet_article.py` + `api/coach_diet_article.py` + `api/admin_diet_article.py`。
三个路由文件对应三个角色，权限通过 `get_current_user` 依赖注入校验。

**3. 前端要做什么：**
- 用户端：饮食中心页面，展示文章卡片列表（支持分类筛选 + 关键词搜索 + 分页），点击进入文章详情页
- 教练端：我的投稿列表、发布文章表单（含封面上传 + 富文本内容）、修改被驳回文章、删除待审核/已驳回文章
- 管理员端：文章审核列表（可按状态筛选）、查看文章详情、通过/驳回/下架操作

**4. 后端要做什么：**
- `schemas/diet_article.py`：ArticleCreate / ArticleUpdate / ArticleReview 等 Pydantic 模型
- `CRUD/diet_article.py`：增删改查 + 审核 + 下架操作
- `api/diet_article.py`：用户端只读接口（无需登录）
- `api/coach_diet_article.py`：教练端 CRUD（需 coach 角色）
- `api/admin_diet_article.py`：管理员审核接口（需 admin 角色）
- `main.py`：注册三个新路由

**5. 接口设计：**

用户端（无需登录）：
```
GET  /api/diet/articles              文章列表（status=1，支持分类筛选+关键词+分页）
GET  /api/diet/articles/{id}         文章详情（浏览数+1）
```

教练端（需 coach 角色）：
```
POST   /api/coach/diet/articles/upload/cover   上传封面图
POST   /api/coach/diet/articles                发布文章（status=0 待审核）
GET    /api/coach/diet/articles                我的投稿列表
PUT    /api/coach/diet/articles/{id}           修改被驳回的文章（重置为待审核）
DELETE /api/coach/diet/articles/{id}           删除待审核/已驳回的文章
```

管理员端（需 admin 角色）：
```
GET  /api/admin/diet/articles                        全部文章（可按 status 筛选，分页）
GET  /api/admin/diet/articles/{id}/detail            单条完整详情（不限状态，不增浏览数）
PUT  /api/admin/diet/articles/{id}/review            审核（通过/驳回）
PUT  /api/admin/diet/articles/{id}/offline           下架已上线文章
```

**6. 数据表结构：**
已有 `diet_articles` 表（`models/diet.py` 中 `DietArticle` 类），字段完整，无需新增。

```
status: 0=待审核  1=已上线  2=已驳回  3=已下架
```

## 一、数据模型（已有，无需修改）

`models/diet.py` → `DietArticle`

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | Integer (PK) | 主键，BaseModel 自动生成 |
| `author_id` | Integer | 投稿教练的 user_id |
| `title` | String(200) | 文章标题 |
| `category` | String(30) | 分类：增肌餐 / 减脂餐 / 均衡饮食 / 补剂知识 |
| `cover_img` | String(500) | 封面相对路径，默认 `""` |
| `content` | Text | 文章正文（富文本 HTML） |

| `status` | SmallInteger | 0=待审核 1=已上线 2=已驳回 3=已下架 |
| `reject_reason` | String(255) | 驳回原因，默认 `""` |
| `reviewed_by` | Integer | 审核人 user_id，可为 null |
| `reviewed_at` | DateTime | 审核时间，可为 null |
| `is_deleted` | SmallInteger | 软删标记（BaseModel 提供），0=正常 1=已删 |
| `created_at` | DateTime | 创建时间（BaseModel 提供） |
| `updated_at` | DateTime | 更新时间（BaseModel 提供） |

## 二、Schemas 设计

文件：`schemas/diet_article.py`

### ArticleCreate（教练发布）

| 字段 | 类型 | 必填 | 默认 | 说明 |
|------|------|:----:|------|------|
| `title` | str | 是 | — | 文章标题，1~200 字符 |
| `category` | str | 是 | — | 分类，枚举：增肌餐/减脂餐/均衡饮食/补剂知识 |
| `cover_img` | str | 否 | `""` | 封面相对路径，由上传接口返回 |
| `content` | str | 否 | `""` | 正文 HTML |
| `summary` | str | 否 | `""` | 摘要，最多 500 字符 |

### ArticleUpdate（教练修改被驳回文章，字段均可选）

| 字段 | 类型 | 说明 |
|------|------|------|
| `title` | str \| None | 标题 |
| `category` | str \| None | 分类 |
| `cover_img` | str \| None | 封面 |
| `content` | str \| None | 正文 |
| `summary` | str \| None | 摘要 |

### ArticleReview（管理员审核）

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `status` | int | 是 | 仅允许 1（通过）或 2（驳回） |
| `reject_reason` | str | 条件 | 驳回时必填且不能为空 |

## 三、CRUD 层设计

文件：`CRUD/diet_article.py`

| 函数 | 说明 |
|------|------|
| `get_articles(db, category, keyword, page, page_size)` | 用户端列表：status=1，未软删，按浏览量降序 |
| `get_article_by_id(db, article_id)` | 按 ID 查单条（未软删），不限状态 |
| `increment_view(db, article)` | 浏览数 +1 |
| `create_article(db, author_id, data)` | 创建文章，status=0 |

| `soft_delete_article(db, article)` | 软删除：is_deleted=1 |
| `get_admin_articles(db, status, page, page_size)` | 管理员列表，可按 status 筛选，按创建时间降序 |
| `review_article(db, article, status, reject_reason, admin_id)` | 审核：更新 status/reject_reason/reviewed_by/reviewed_at |
| `offline_article(db, article)` | 下架：status=3 |

## 四、API 路由设计

### 4.1 用户端 `api/diet_article.py`

路由前缀：`/diet/articles`，挂载后完整路径：`/api/diet/articles`

**鉴权**：无需登录

#### GET `/api/diet/articles` — 文章列表

Query 参数：

| 参数 | 类型 | 必填 | 默认 | 说明 |
|------|------|:----:|------|------|
| `category` | string | 否 | — | 分类精确匹配 |
| `keyword` | string | 否 | — | 标题模糊搜索 |
| `page` | integer | 否 | `1` | 页码 |
| `page_size` | integer | 否 | `12` | 每页条数 |

成功 `data`：

```json
{
  "list": [
    {
      "id": 1,
      "title": "增肌期饮食指南",
      "category": "增肌餐",
      "cover_img": "diet_covers/xxx.jpg",
      "summary": "...",
      "view_count": 128,
      "author_name": "张教练",
      "created_at": "2026-04-13 10:00:00"
    }
  ],
  "total": 50,
  "page": 1,
  "page_size": 12
}
```

#### GET `/api/diet/articles/{article_id}` — 文章详情

每次成功返回前 `view_count +1`。

成功 `data`：

```json
{
  "id": 1,
  "title": "增肌期饮食指南",
  "category": "增肌餐",
  "cover_img": "diet_covers/xxx.jpg",
  "content": "<p>正文 HTML...</p>",
  "summary": "...",
  "view_count": 129,
  "author_name": "张教练",
  "created_at": "2026-04-13 10:00:00"
}
```

业务错误：文章不存在、未上线或已软删 → **404**，`message` 为「文章不存在」。

### 4.2 教练端 `api/coach_diet_article.py`

路由前缀：`/coach/diet/articles`，挂载后完整路径：`/api/coach/diet/articles`

**鉴权**：需 JWT，`role == "coach"`；非教练返回 **403**，`message` 为「无权限」。

#### POST `/api/coach/diet/articles/upload/cover` — 上传封面图

- `Content-Type: multipart/form-data`，表单字段 `file`
- 限制：MIME 须为 `image/jpeg`、`image/png`、`image/webp`；不超过 **5MB**
- 写入 `static/diet_covers/`，文件名：`{user_id}_{uuid}{后缀}`

成功 `data`：

```json
{ "path": "diet_covers/1_abc123.jpg" }
```

完整访问 URL：`{服务根}/api/static/diet_covers/1_abc123.jpg`

#### POST `/api/coach/diet/articles` — 发布文章

请求体：`ArticleCreate`，初始 `status=0`（待审核）

成功 `data`：`{ "id": 新建文章ID }`

#### GET `/api/coach/diet/articles` — 我的投稿列表

返回当前教练未软删的全部投稿，按创建时间降序。

成功 `data`（数组）：

| 字段 | 说明 |
|------|------|
| `id` | 文章 ID |
| `title` | 标题 |
| `category` | 分类 |
| `cover_img` | 封面相对路径 |
| `summary` | 摘要 |
| `status` | 0 待审核 / 1 通过 / 2 驳回 |
| `reject_reason` | 驳回原因，未驳回时为 `""` |
| `created_at` | 创建时间 |

#### PUT `/api/coach/diet/articles/{article_id}` — 修改被驳回文章

- 仅限**本人**且 `status == 2`（已驳回）时可修改
- 保存后重置 `status=0`，清空 `reject_reason`

请求体：`ArticleUpdate`（字段均可选，只传需修改的）

成功：`data: null`，`message: "已重新提交审核"`

业务错误：非本人或不存在（404）、状态不是已驳回（400）

#### DELETE `/api/coach/diet/articles/{article_id}` — 删除文章

- 仅限**本人**且 `status` 为 **0（待审核）** 或 **2（已驳回）** 时可删
- 执行软删除（`is_deleted=1`）

成功：`data: null`，`message: "已删除"`

业务错误：非本人或不存在（404）、已通过等不可删状态（400）

### 4.3 管理员端 `api/admin_diet_article.py`

路由前缀：`/admin/diet/articles`，挂载后完整路径：`/api/admin/diet/articles`

**鉴权**：需 JWT，`role == "admin"`；非管理员返回 **403**，`message` 为「无权限」。

#### GET `/api/admin/diet/articles` — 文章列表

Query 参数：

| 参数 | 类型 | 必填 | 默认 | 说明 |
|------|------|:----:|------|------|
| `status` | integer | 否 | — | 0/1/2/3；不传返回全部未软删 |
| `page` | integer | 否 | `1` | 页码 |
| `page_size` | integer | 否 | `20` | 每页条数 |

成功 `data`：

```json
{
  "list": [
    {
      "id": 1,
      "title": "增肌期饮食指南",
      "category": "增肌餐",
      "status": 0,
      "reject_reason": "",
      "author_id": 5,
      "author_name": "张教练",
      "created_at": "2026-04-13 10:00:00"
    }
  ],
  "total": 30,
  "page": 1,
  "page_size": 20
}
```

#### GET `/api/admin/diet/articles/{article_id}/detail` — 单条详情（预览）

- 不限状态（待审核/已驳回/已下架均可查）
- **不增加** `view_count`

成功 `data`：

```json
{
  "id": 1,
  "title": "增肌期饮食指南",
  "category": "增肌餐",
  "cover_img": "diet_covers/xxx.jpg",
  "content": "<p>正文...</p>",
  "summary": "...",
  "view_count": 128,
  "status": 0,
  "reject_reason": "",
  "author_name": "张教练",
  "created_at": "2026-04-13 10:00:00"
}
```

业务错误：不存在或已软删 → **404**

#### PUT `/api/admin/diet/articles/{article_id}/review` — 审核

- 仅当 `status == 0`（待审核）时可审核
- 通过：`status=1`；驳回：`status=2` + 写入 `reject_reason`
- 记录 `reviewed_by`（当前管理员 ID）和 `reviewed_at`

请求体：`ArticleReview`

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `status` | int | 是 | 仅允许 1 或 2 |
| `reject_reason` | str | 条件 | 驳回时必填且不能为空 |

成功：`data: null`，`message: "已通过"` 或 `"已驳回"`

业务错误：`status` 非 1/2（400）、驳回未填原因（400）、不存在（404）、当前非待审核（400）

#### PUT `/api/admin/diet/articles/{article_id}/offline` — 下架

- 仅当 `status == 1`（已上线）时可下架
- 将 `status` 改为 `3`（已下架），用户端不可见

成功：`data: null`，`message: "已下架"`

业务错误：不存在（404）、当前非已上线（400）

## 五、文章状态流转

```
教练投稿
    ↓
[0] 待审核
    ↓ 管理员审核
   ↙         ↘
[1] 已上线   [2] 已驳回
    ↓              ↓ 教练修改后重新提交
[3] 已下架      [0] 待审核（循环）
```

| status | 含义 | 用户端可见 | 教练可修改 | 教练可删除 |
|:------:|------|:----------:|:----------:|:----------:|
| 0 | 待审核 | 否 | 否 | 是 |
| 1 | 已上线 | 是 | 否 | 否 |
| 2 | 已驳回 | 否 | 是 | 是 |
| 3 | 已下架 | 否 | 否 | 否 |

## 八、补充说明

1. **静态资源**：封面图存于 `static/diet_covers/`，访问路径为 `{服务根}/api/static/diet_covers/{文件名}`，与动作库封面访问方式一致。
2. **富文本内容**：`content` 字段存储 HTML 字符串，前端使用 `v-html` 渲染，注意 XSS 防护（可在后端做 HTML 净化或前端使用 DOMPurify）。
3. **分类枚举**：`category` 建议在 schemas 层用 `Literal` 或 `Enum` 约束为：`增肌餐`、`减脂餐`、`均衡饮食`、`补剂知识`。
4. **教练修改流程**：先可选调用 `upload/cover` 获取新封面路径，再 `PUT` 提交修改。
5. **与动作库的差异**：文章无视频上传、无 `steps`/`cautions` 字段；新增 `summary`（摘要）和 `content`（富文本正文）字段。
