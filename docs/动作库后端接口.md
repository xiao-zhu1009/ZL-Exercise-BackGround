【任务书】

1. 需求说明：
   实现动作库模块的完整后端接口，支持教练投稿、超管审核、用户浏览三条业务线。

2. 实现思路：
   分三层：schemas（数据校验）→ CRUD（数据库操作）→ api（路由）。
   三个路由文件对应三个角色，权限通过 get_current_user 依赖注入校验。

3. 前端要做什么：
   取消 src/api/action.js 中的 mock，改为真实 axios 请求。

4. 后端要做什么：
   - schemas/action.py：ActionCreate / ActionUpdate / ActionReview
   - CRUD/action.py：增删改查 + 审核操作
   - api/action.py：用户端只读接口（无需登录）
   - api/coach_action.py：教练端 CRUD（需 coach 角色）
   - api/admin_action.py：管理员审核接口（需 admin 角色）
   - main.py：注册三个新路由

5. 接口设计：
   用户端：
     GET  /api/actions                  列表（status=1，支持筛选+分页）
     GET  /api/actions/{id}             详情（浏览数+1）

   教练端：
     POST   /api/coach/actions          发布动作（status=0 待审核）
     GET    /api/coach/actions          我的投稿列表
     PUT    /api/coach/actions/{id}     修改被驳回的动作（重置为待审核）
     DELETE /api/coach/actions/{id}     删除待审核/已驳回的动作

   管理员端：
     GET  /api/admin/actions            全部动作（可按 status 筛选）
     PUT  /api/admin/actions/{id}/review   审核（通过/驳回）
     PUT  /api/admin/actions/{id}/offline  下架已上线动作

6. 数据表结构：
   已有 actions 表，字段完整，无需新增。
   status: 0=待审核 1=已上线 2=已驳回 3=已下架
