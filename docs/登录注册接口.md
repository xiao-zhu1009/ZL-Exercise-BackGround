# 登录与注册接口说明

认证相关路由定义在 `api/auth.py`，经 `main.py` 挂载后**统一前缀**为 `/api`（即完整路径形如 `/api/auth/...`）。

## 统一响应格式

| 场景 | HTTP 状态码 | Body 结构 |
|------|-------------|-----------|
| 成功 | 200 | `{ "code": 200, "message": "提示文案", "data": ... }` |
| 业务失败 | 与 `code` 一致（如 400/401/403） | `{ "code": <数字>, "message": "错误说明", "data": null }` |

请求体均为 **JSON**，`Content-Type: application/json`。

---

## 接口索引

| 序号 | 说明 | 方法与路径 |
|:----:|------|------------|
| 1 | 登录 | `POST` `/api/auth/login` |
| 2 | 发送验证码（注册用手机号） | `POST` `/api/auth/send-code` |
| 3 | 校验验证码 | `POST` `/api/auth/verify-code` |
| 4 | 注册 | `POST` `/api/auth/register` |

---

## 1. 登录

| 项目 | 内容 |
|------|------|
| 路径 | `POST /api/auth/login` |
| 作用 | 校验账号/密码，签发 JWT 并落库；前端后续请求头携带 `Authorization: Bearer <token>` |

**请求体字段**

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `account` | string | 是 | 用户名或手机号 |
| `password` | string | 是 | 密码 |

**成功时 `data`**

| 字段 | 说明 |
|------|------|
| `id` | 用户 ID |
| `username` | 用户名 |
| `role` | 角色 |
| `nickname` | 昵称 |
| `avatar` | 头像 URL，无则为 `""` |
| `token` | JWT |

---

## 2. 发送验证码

| 项目 | 内容 |
|------|------|
| 路径 | `POST /api/auth/send-code` |
| 作用 | 检查手机号是否已注册；未注册则视为可进入注册流程。当前为**测试模式**，不发送真实短信，测试固定码见文末说明 |

**请求体字段**

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `phone` | string | 是 | 手机号 |

**成功时 `data`**：`null`

---

## 3. 校验验证码

| 项目 | 内容 |
|------|------|
| 路径 | `POST /api/auth/verify-code` |
| 作用 | 注册前校验用户填写的验证码是否与后端测试码一致，且手机号未被注册 |

**请求体字段**

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `phone` | string | 是 | 手机号 |
| `code` | string | 是 | 用户输入的验证码 |

**成功时 `data`**：`null`

---

## 4. 注册

| 项目 | 内容 |
|------|------|
| 路径 | `POST /api/auth/register` |
| 作用 | 再次校验验证码后，检查用户名与手机号唯一性，创建新用户 |

**请求体字段**

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `phone` | string | 是 | 手机号 |
| `code` | string | 是 | 验证码（须与测试码一致） |
| `username` | string | 是 | 用户名 |
| `password` | string | 是 | 密码 |
| `nickname` | string | 否 | 昵称 |

**成功时 `data`**

| 字段 | 说明 |
|------|------|
| `id` | 新用户 ID |
| `username` | 用户名 |

---

## 补充说明

1. **测试验证码**：当前后端常量固定为 `123456`（见 `api/auth.py` 中 `FAKE_CODE`），正式环境需替换为真实短信服务后再改逻辑。
2. **登录 `account`**：与数据库查询一致，支持**用户名或手机号**作为登录账号字段。
3. **注册流程建议**：`send-code` → 用户收码（测试下为固定码）→ `verify-code` → `register`（`register` 内仍会再次校验 `code`）。
4. **OAuth2 文档**：其他需登录接口使用的 `OAuth2PasswordBearer` 的 `tokenUrl` 为 `/auth/login`（OpenAPI 展示用）；实际调用登录接口的完整路径仍为 **`POST /api/auth/login`**。
