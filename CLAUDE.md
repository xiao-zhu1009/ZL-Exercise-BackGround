# 项目长期规则（Claude 必须严格遵守）

## 一、项目技术栈（固定不变）
前端：
- Vue2 + JavaScript
- Element-UI
- Axios 请求接口
- 单页面 SPA 开发

后端：
- FastAPI
- 异步 SQLAlchemy ORM
- MySQL 数据库
- 异步数据库操作

接口规范：
- 统一返回格式：{ code: 200, msg: "提示", data: 数据 }
- 错误返回：{ code: 错误码, msg: "错误信息", data: null }

## 二、编码规范（必须遵守）
1. 前端使用 Vue2 选项式 API，不要用 Vue3
2. 前端组件结构清晰，data、methods、created 分开写
3. 接口请求统一使用 axios
4. 后端路由清晰、注释完整
5. 数据库操作全部使用异步 async/await
6. 所有接口必须做异常处理
7. 代码简洁、易读、不冗余

## 三、最重要的强制规则：每次必须先出【任务书】
你在生成任何代码之前，**必须先输出任务书并放在对应文件夹下的docs下**，格式如下：

--------------------------------
【任务书】
1. 需求说明：xxx
2. 实现思路：xxx
3. 前端要做什么：xxx
4. 后端要做什么：xxx
5. 接口设计：xxx
6. 数据表结构（如果需要）：xxx
--------------------------------

然后再开始写代码！

## 四、回答格式要求
1. 先任务书
2. 再前端代码
3. 再后端代码
4. 最后说明怎么运行

## 五、禁止事项
- 禁止使用 Vue3
- 禁止使用 TypeScript
- 禁止使用同步数据库操作
- 禁止不写任务书
